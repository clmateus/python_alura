# Bancos Relacionais

Um banco de dados relacional é um sistema de armazenamento onde os dados são organizados em tabelas (também chamadas de relações). A principal ideia é que os dados sejam armazenados de forma estruturada, evitando repetições desnecessárias e garantindo a precisão das informações.

Cada tabela é composta por:
- Colunas (Atributos) - Definem o tipo de dado que está sendo armazenado. Por exemplo: Nome, Email, Preço, etc.
- Linhas (Registros ou Tuplas) - São os dados em si. Cada linha representa uma entrada única na tabela. Por exemplo: Os dados de um cliente específico.

A grande vantagem desse modelo é a capacidade de cruzar informações entre diferentes tabelas de forma rápida e segura.

## Chave Primária (Primary Key / PK)

A Chave Primária é o identificador único de um registro dentro de uma tabela. Ela garante que não existam duas linhas exatamente iguais.

As características principais de uma chave primária são:
- Unicidade: O valor da chaveprimária nunca pode se repetir na mesma tabela.
- Não Nula (Not Null): Um registro não pode existir sem uma chave primária, ela nunca pode ficar em branco.

Exemplo: Numa tabela de clientes pode haver várias pessoas com o mesmo nome, mas a coluna ID_Cliente ou CPF será única para cada uma.

## Chave Estrangeira (Foreign Key - FK)

A Chave Estrangeira é o mecanismo que usamos para conectar duas tabelas. Basicamente, ela é uma coluna em uma tabela que faz referência direta à Chave Primária de outra tabela.

A função da chave estrangeira é garantir a integridade referencial. Isso significa que o banco de dados não permitirá que você crie um vínculo com algo que não existe (por exemplo, registrar um pedido para um cliente que não está cadastrado).

Exemplo: Numa tabela de pedidos existe uma coluna chamada ID_Cliente. Essa coluna é uma chave estrangeira que aponta para a chave primária de uma tabela Clientes. Assim, sabemos exatamente quem fez aquele pedido.

## Relacionamentos

Os relacionamentos definem como as tabelas interagem entre si através das chaves primárias e estrangeiras. Existem três tipos principais de relacionamentos lógicos em bancos relacionais:

- 1:1 - Um para um

Um registro da tabela A se relaciona com apenas um registro da tabela B, e vice-versa.

- 1:N - Um para muitos

Um registro da tabela A pode se relacionar com vários registros da tabela B, mas o registro da tabela B se relaciona apenas com um da tabela A.

- N:N - Muitos para muitos

Um registro da tabela A pode se relacionar com vários da tabela B, e vice-versa. Bancos relacionais não suportam isso diretamente, mas é possível fazer isso através da criação de uma tabela associativa (ou tabela de junção) no meio.

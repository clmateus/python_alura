## O que é uma Requisição (Request)?

Continuando com a nossa analogia do restaurante, a **requisição** é o momento exato em que você faz o seu pedido ao garçom. No mundo da tecnologia, é a mensagem que um programa (o cliente) envia para outro (o servidor) pedindo para realizar uma ação.

Sempre que você acessa um site, curte uma foto ou envia uma mensagem no chat, o seu aplicativo está, nos bastidores, enviando uma requisição.

### As Partes de uma Requisição

Para que o servidor entenda perfeitamente o que você precisa, uma requisição é dividida em quatro partes principais:

*   **Endpoint / URL (O Endereço):** Indica para onde a mensagem está sendo enviada e qual recurso você quer acessar. 
*   **Método (A Ação):** É o verbo que diz o que você quer fazer. Os mais comuns são:
    *   **GET:** "Me traga uma informação" (Ex: carregar a previsão do tempo).
    *   **POST:** "Crie algo novo" (Ex: enviar um formulário ou publicar um tweet).
    *   **PUT / PATCH:** "Atualize uma informação" (Ex: editar a sua foto de perfil).
    *   **DELETE:** "Apague algo" (Ex: deletar um e-mail).
*   **Headers (Os Cabeçalhos):** São informações extras que vão "escondidas" na mensagem. Eles dizem, por exemplo, qual é o tipo de dispositivo que você está usando ou enviam uma chave de segurança para provar que você está logado.
*   **Body (O Corpo):** É o conteúdo da sua requisição. Se você está fazendo um `POST` para criar uma conta nova, o *body* é a parte que contém o seu nome, seu e-mail e a sua senha.

Em resumo: a requisição é a pergunta ou o comando que você manda para a API. Assim que o servidor processa isso, ele devolve uma **Response** (Resposta), dizendo se deu tudo certo e trazendo os dados que você pediu.
